# Дизайн-бриф — Aadhaar-этап через единый компонент (`AadhaarScanPanel`)

> Автор: Ульяна (дизайн-лид, клиентская сторона). Для Феди (код). Сценарии: Компания + общий v2.
> Утверждено Денисом: унифицировать 3 разъехавшихся Aadhaar-экрана одним общим компонентом — по образцу VKYC → `VideoIdentPanel`.
> Эталон порядка (Денис подтвердил) = вход компании. Регуляторика (Марго): «нельзя Aadhaar без согласия».

## TL;DR

- Сейчас 3 Aadhaar-экрана разъехались: разный порядок (QR до/после согласий), разный QR (`QrSvg` vs `QrMock`), ИП вообще не показывает 5 полей данных и дублирует код «How it works».
- Закрепляем **один канонический порядок** и выносим в **общий презентационный компонент `AadhaarScanPanel`** (как `VideoIdentPanel`): он держит порядок, QR-замок, скан, спиннер, блок данных. Логику (API `passXAadhaar`, навигация) экраны оставляют у себя.
- Что параметризуется: **набор согласий до QR** (1–2, по контексту — это ОК), **слот после данных** (реестры / ничего / своё), `variant` для текста «How it works».
- Что строго одинаково: порядок, **один `QrSvg`** (убрать `QrMock`), **5 полей после скана везде** (включая ИП — добавить), ссылка «Скачать приложение», маскировка номера.
- Ограничение «один CodeField/интерактив на экран» — про пин (DSC OTP), Aadhaar не нарушает: у Aadhaar нет CodeField.

## Канонический порядок Aadhaar-этапа (сверху вниз)

Один и тот же на всех трёх экранах:

1. **Заголовок + подзаголовок** шага (рисует экран — `CardHeader`).
2. **«How it works»** — `AadhaarHowTo` (вариант по контексту), ДО QR. Успокаивающая строка + шаги 1-2-3 + «Подробнее» (модалка).
3. **Ссылка «Скачать приложение Aadhaar»** — тихой строкой рядом с инструкцией (нейтральная, `textSecondary`, не accent — у пользователя может не быть приложения).
4. **Согласия** — набор по контексту (Checkbox в `ConsentRow`), с лид-строкой «перед идентификацией подтвердите согласия». ГЕЙТЯТ QR.
5. **QR** — ПОД ЗАМКОМ до всех согласий (`QrLocked`: иконка-замок + подсказка «дайте согласия выше»), разблокируется в `QrBlock` (`QrSvg` в белом фрейме + caption) после согласий.
6. **Кнопка «Я отсканировал код»** (`view="accent"`, справа). Ряд с «Назад»/«Выйти» — по контексту экрана.
7. **Спиннер** «Получаем данные из UIDAI…» (~2 с).
8. **Блок данных** — `AadhaarResultBox`: 5 полей (Имя, Номер Aadhaar *маскированный*, Телефон, Email, Адрес) + success-плашка над ним. **Везде, включая ИП.**
9. **Контекстное продолжение** — слот после данных (реестры / ничего / своё) + кнопка «Продолжить».

Маркер необратимости (`IrreversibleMarker`, сине-серый, не warning) после скана — оставляем на усмотрение экрана (есть в сессии подписанта), компонент его не навязывает.

## API общего компонента — `AadhaarScanPanel`

Презентационный (как `VideoIdentPanel`): чистая презентация, без API-вызовов и навигации. Фаза приходит пропом, тексты — пропами (двуязычность у экрана).

```
type AadhaarPhase = 'qr' | 'waiting' | 'success' | 'error';

interface AadhaarConsent {
  id: string;            // ключ для экрана (eKYC / privacy)
  label: string;
  description: string;
  checked: boolean;
  onChange: (checked: boolean) => void;
}

interface AadhaarScanPanelProps {
  phase: AadhaarPhase;
  variant: 'entry' | 'session';   // → AadhaarHowTo
  consents: AadhaarConsent[];      // набор согласий ДО QR (1..n); гейтят QR
  showAppLink?: boolean;           // дефолт true (ссылка «Скачать приложение»)
  appLinkHref?: string;            // дефолт — UIDAI get-aadhaar

  result?: AadhaarResult | null;   // данные после скана → AadhaarResultBox (5 полей)

  onScanned: () => void;           // клик «Я отсканировал» (экран запускает passXAadhaar)

  // тексты (двуязычность задаёт экран — паттерн VideoIdentPanel)
  texts: {
    consentLead: string;
    qrLockedHint: string;
    qrCaption: string;
    appLink: string;
    ctaScanned: string;
    waiting: string;
    success: string;
    errorTitle?: string; errorText?: string; retry?: string;
  };

  slotAfterData?: React.ReactNode; // контекстное продолжение под данными (реестры / кнопки / ничего)
  backSlot?: React.ReactNode;      // «Назад»/«Выйти» в ряду с «Я отсканировал» (по контексту)
  demoFailSlot?: React.ReactNode;  // демо-леса (имитировать ошибку) — опционально
}
```

Внутренняя логика компонента:
- QR заблокирован, пока не все `consents[].checked === true` → `QrLocked`; иначе `QrBlock` с `QrSvg`.
- `phase === 'waiting'` → спиннер; `'success'` → success-плашка + `AadhaarResultBox` (если `result`) + `slotAfterData`; `'error'` → `Note negative` + «Повторить».
- Кнопка «Я отсканировал» активна только при всех согласиях; в ряду — `backSlot` слева, accent справа.

Что ОСТАЁТСЯ у экранов (не в компоненте):
- `useState` фазы, `result`, флаги согласий.
- API: `passCompanyAadhaar` / `passSignatoryAadhaar` / `giveConsent('Aadhaar')` и т.п.
- Навигация (`navigate`), гейт «Продолжить», запись шага.
- Слот после данных целиком (реестры на входе компании — это `Checkbox` + кнопка, рисует экран).

Имя: **`AadhaarScanPanel`** (рекомендую) — симметрично `VideoIdentPanel`. Альтернатива `AadhaarStep` хуже: «Step» намекает на роутинг/логику шага, а компонент презентационный.

SDDS-компоненты внутри: `Checkbox` (согласия), `Button` (скан/повтор), `Note view="negative"` (ошибка). `AadhaarHowTo`, `AadhaarResultBox`, `QrSvg` — уже наши общие, переиспользуем. `QrMock` удалить.

## Edge-cases / состояния

- **error** (скан не прошёл): `Note negative` + «Повторить скан». Сейчас только у входа компании; в компоненте — для всех, демо-триггер опционален (`demoFailSlot`).
- **0 согласий дано** → QR под замком, кнопка disabled.
- **back/exit**: вход компании — нет (первый шаг); подписант — «Выйти/К дашборду»; ИП — «Назад». Через `backSlot`.
- **a11y**: замок-иконка `aria-hidden`, QR `role="img" aria-label`; видимый фокус на Checkbox/Button (SDDS).

## Что проверить (mobile + desktop)

- Порядок блоков идентичен на всех 3 экранах; ширина 880 (`ScreenV2`).
- QR реально под замком до согласий; разблокировка по последнему чекбоксу.
- 5 полей данных появляются после скана на ВСЕХ трёх (в т.ч. ИП), номер маскирован.
- Ссылка «Скачать приложение» видна сразу, тихая, не конкурирует с accent-кнопкой.
- Один accent-CTA на экране (правило иерархии).
- Mobile: согласия и QR-блок не ломают ритм 24px; модалка «Подробнее» скроллится.
