## О чём встреча

Рабочая сессия по правильной последовательности экранов клиентского пути. Марго показала верхнеуровневую карту прототипа, прошлись по каждому шагу: что переносить, убирать, добавлять. Темы: место и состав consent-экранов, сбор директоров/подписантов через Probe42-дропдаун, правила выбора подписантов и authorized signatories, логика дашборда «что ещё не готово» для мульти-VCIP, поведение экрана подтверждения (edit + загрузка proof). Фокус — happy-path (STP) с данными из Probe42.

## Ключевые решения

- **Скоуп прототипа: один тип юрлица, happy path (STP).** Ручной ввод (нет данных из Probe42) — откладываем.
- **Порядок flow:** Лендинг → Self-registration (email) → email-письмо со ссылкой → PAN → вопросы о роли → выбор директоров-подписантов из дропдауна (Probe42) → выбор authorized signatories → BNQ (индустрия, PEP, FATCA) → Consents → Подтверждение данных компании (с Edit) → VCIP (каждый подписант) → подписание Board Resolution → финал (дашборд счёта).
- **Consent — два слоя, две точки:** (1) перед PAN — согласие на сбор данных из индийских реестров; (2) перед VCIP — privacy notice + согласие с достоверностью анкеты.
- **Риск-категория клиенту не показывается** — внутренняя логика банка, убрать с клиентских экранов.
- **Экран подтверждения = view + edit.** Edit → поля + под каждое изменённое поле запрос proof-документа. Изменённое уходит на ручной ревью (DVU), счёт не активен до проверки.
- **Директора/подписанты — дропдаун из Probe42** (тянем по PAN), не ручной ввод. Клиент выбирает: кто подписывает BR (мин. 2 директора или 1 секретарь), кто — authorized signatory (из директоров или новый).
- **VCIP — каждый подписант отдельно** (мульти-логин в одну сессию компании).
- **После VCIP** — подписание Board Resolution и анкеты в той же сессии.
- **Дашборд прогресса** — не линейный степпер, а живая карта статусов: что не закрыто, у кого не пройден VCIP, какие запросы документов от DVU. Маркер «Requires Action». Ориентир — Revolut.
- **Лендинг** — без продуктовой привязки («счёт за 1 день» убрать), просто онбординг. Вставить вариант Марго.
- **Email-письмо** (ссылка-приглашение) — добавить наглядным экраном.
