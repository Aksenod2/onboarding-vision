## Общие правила

- **Layout:** каждый экран (кроме лендинга SP-01, у него своя расширенная шапка) оборачивается в `<ScreenV2>` — общая шапка (логотип «SBER Банк» + RU/EN) + контейнер.
- **Язык:** `useLanguage()` → `{ lang, setLang }`; локальный `dict: Record<Lang, {...}>` в каждом экране.
- **Данные:** только через `app/src/mock/v2/api.ts` (не лезть в seed напрямую). Загрузка с задержкой 350ms → показывать состояния.
- **SDDS:** Button, TextField, CodeField, Select, Checkbox, Notification, Steps и т.д. — свериться с MCP `sdds-serv` перед использованием; если недоступен — по образцу + `// TODO свериться с MCP`.
- **Риск-скор НЕ показываем** клиенту нигде.
- **Служебные кнопки** — неакцентные (view secondary/clear). Главный CTA — accent.
- **Переходы:** `navigate('/v2/...')` по карте маршрутов ниже.

## SP-03 · Регистрация / авторизация + согласия
- **Цель:** проспект создаёт вход (email + телефон → OTP) и даёт стартовые согласия.
- **Шаг 1 (ввод):**
  - `TextField` email; `TextField` телефон (+91, маска); подпись-помощь.
  - **3 согласия чекбоксами:** Cookie (необяз.), Terms & Conditions 2b (обяз.), Privacy Notice (обяз.) — короткие лейблы, ключевые слова как ссылки (Privacy Notice, T&C). Тексты — из [[Consents — список (current)]] (1, 2b, 3).
  - `Button` accent «Получить код» — активна, когда обязательные согласия отмечены и поля заполнены.
  - API: `createSession(email, phone, lang)` + `giveConsent('Terms & Conditions'|...)` с timestamp.
- **Шаг 2 (OTP):** `CodeField` 4 знака; подсказка «код 0000 для демо»; `Button` «Подтвердить». API: `verifyOtp(code)` → ok при `0000`.
- **Успех →** `/v2/registry` (SP-04).
- **Состояния:** загрузка кнопки, ошибка неверного кода (`Notification` или inline).

## SP-04 · Consent: доступ к индийским реестрам
- **Цель:** согласие на запрос данных из реестров ДО ввода PAN (BR-01).
- **Композиция:** заголовок + поясняющий текст (зачем доступ: PAN, Probe42, CKYC), `Checkbox`/`Notification` с согласием, `Button` accent «Разрешить и продолжить».
- **API:** `giveConsent('Registry Access', timestamp)`.
- **→** `/v2/pan` (SP-05).

## SP-05 · Ввод PAN + авто-скрининг
- **Цель:** клиент вводит PAN → запуск авто-скрининга (компания + лицо «Стоп-42»).
- **Шаг 1:** `TextField` PAN (формат `AAAAA9999A`, аптокейс), подсказка. `Button` «Проверить». Демо-значение: `ABFPS4321K`.
- **Шаг 2 (скрининг):** анимированный список проверок из `getScreening()`: PAN, OFAC/Sanctions, Probe42, CKYC, **Stop-42** — каждая загорается Pass/Fetched по очереди (имитация). Текст «Подтягиваем данные из реестров…».
- **→** авто-переход `/v2/company` (SP-06) по завершении.
- **API:** `getScreening()`, `getBusiness()`.

## SP-06 · Подтверждение данных компании (+ Edit/proof)
- **Цель:** показать подтянутые Probe42-данные; дать правку с загрузкой proof.
- **Режим View:** карточка данных бизнеса из `getBusiness()` — поля с бейджем «из реестра» (по `businessFieldSources`: registry = read-only + бейдж; manual = редактируемо). Поля: tradeName, pan, gstin, udyam, commencementDate, registeredAddress, industry/segment, companyResidency.
- **Режим Edit:** `Button` «Изменить» → поля становятся `TextField`; **под каждым изменённым полем появляется блок загрузки proof-документа**; `Notification`: «Изменённые данные уйдут на проверку (DVU), счёт активируется после проверки». API: `updateBusiness(patch)`, `uploadDocument`.
- **CTA:** «Подтвердить и продолжить» → `/v2/bnq` (SP-07).
- **Важно:** реализовать оба состояния (view/edit) как переключение.

## SP-07 · BNQ-опросник (Q1–Q11)
- **Цель:** пошаговый опросник, двойной сценарий каждого вопроса. Самый крупный экран.
- **Данные:** `getBnq()` → массив из 11 ответов (q, attribute, source, value, riskScore). Управляй опросником по этому массиву.
- **Логика вопроса:**
  - `source: 'available'` (Probe42 подтянул) → показать «Мы определили: **{value}**. Верно?» + кнопки Да / Изменить.
  - `source: 'not_available'` → задать вопрос с вариантами (`Select`/радио/`TextField`).
- **Спец-шаги (по [[BNQ Template A — Sole Proprietorship]]):**
  - Q1: индустрия+сегмент → **загрузка business licence** (`uploadDocument('Business Licence')`).
  - Q5 PEP: «Yes» → пометка алерта DVU (визуально не пугать клиента).
  - Q9 ВЭД: «No» → пропустить Q10/Q11. «export/import/both» → Q10 (страны) → Q11.
  - Q11: дисклеймер про **IEC** + выбор «Upload Now / Upload Later» (`uploadDocument('IEC')`).
  - Consents по ходу: Aadhaar, Data Principals, KMP Confirmation — чекбоксы при сборе данных лица (`giveConsent`).
- **Риск-скор НЕ показываем.** Прогресс-бар вопросов ок (Q3/Q11), но без баллов.
- **API:** `getBnq()`, `answerBnq(q, value)`, `uploadDocument`, `giveConsent`.
- **→** `/v2/pre-vcip` (SP-08).
- **Реализация:** допустимо упростить до пошагового флоу (один вопрос на экране или аккордеон), но пройти все Q1–Q11 с ветвлением Q9.

## SP-08 · Consent перед VCIP
- **Цель:** privacy notice + подтверждение достоверности (BR-02 + Consent 7 «Data Accuracy»).
- **Композиция:** сводка-напоминание + текст согласия (достоверность данных) + `Checkbox` + `Button` accent «Перейти к видеоидентификации».
- **API:** `giveConsent('Data Accuracy', timestamp)`.
- **→** `/v2/vcip` (SP-09).

## SP-09 · VCIP + подписание
- **Цель:** видеоидентификация (один владелец) + подписание декларации/CAF в рамках сессии.
- **Композиция:**
  - Карточка «видеосессия» — плейсхолдер камеры (styled, без реального видео), имя из `getVcip().personName`.
  - Шаги: 1) «Начать видеоидентификацию» (mock — имитация прохождения), 2) «Подписать документы» (декларация + Board Resolution-эквивалент).
  - `Button` «Начать» → имитация → «Подписать и завершить».
- **API:** `getVcip()`, `passVcip()`.
- **→** `/v2/dashboard` (SP-10).

## SP-10 · Дашборд прогресса
- **Цель:** живая карта статусов с «Requires Action» (НЕ финальный экран — постоянный интерфейс). Ориентир — Revolut.
- **Данные:** `getProgress()` → массив шагов (id, title, status: done/current/pending/requires_action, note).
- **Композиция:**
  - Заголовок «Ваша заявка» + общий статус.
  - Список/карточки шагов: иконка статуса (✓ done, ● current, ○ pending, ⚠ requires_action). Шаги `requires_action` — **выделены акцентом**, с `note` и кнопкой действия (напр. «Пройти видеоидентификацию» → `/v2/vcip`).
  - Блок «Что требуется от вас» наверху, если есть requires_action.
- **API:** `getProgress()`, `getCase()`, `setStepStatus`.
