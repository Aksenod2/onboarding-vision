# Компания — аудит дыр (полнота сценария)

> Дата: 2026-06-16. Срез: UX-полнота (Ульяна) + сверка с BRD (Боря).
> Планка (решение Дениса): прототип = **интерактивный продукт** (у пользователя есть агентность), а не сценарный нарратив по одной золотой записи.
> Статус: карта для приоритизации. НЕ чинено.

## Корневая причина (зачем этот документ)

Сценарий Компания построен как **проход по одной золотой записи** (Mehta Textiles): директора предвыбраны из реестра, AS предзаполнен, BR «генерит банк», документы не грузятся. Пока идёшь по рельсам — работает. Шаг вбок (добавить подписанта, загрузить BR) — пусто, потому что для демо-прохода это было не нужно.

Подтверждение масштаба: экраны игнорируют даже данные, которые УЖЕ ЕСТЬ в `companySeed.ts` — массив `bnq[]` (11 вопросов анкеты) и `documents[]` (IEC) нигде не выводятся. То есть зашит нарратив, а не продукт.

## Вопросы к Марго (от Бори)

1. **Добавление AS вручную:** нужна ли кнопка «добавить ещё подписанта» или для пилота фиксируем одного AS?
2. **Директор не в Probe42:** клиент вводит руками или стоп/DVU? (BRD жёстко привязывает валидацию директоров к Probe42, ручной ввод директора не описан.)
3. **BR — два пути:** подтверждаешь обе ветки (шаблон банка И загрузка своего с IDP/OCR)? Загрузка своего = No STP → DVU?
4. **Документы компании:** показываем fallback-загрузку (COI/MOA/AOA/Shareholding/GST) при правке/не подтянулось, как в SP, или happy-path только из Probe42?
5. **UBO/FATCA:** где в потоке заполнителя собираем UBO-декларацию и FATCA-классификацию?

## Рекомендация по умолчанию (Боря)

Критичны к доработке только **#1 (добавление AS)** и **#2 (own-format BR upload)** — прямо названные Денисом и ядро отличия компании. Остальное — собрать в «вне happy-path MVP» и подтвердить scope у Дениса, а не чинить вслепую.

⚠️ Отступление целевки, зафиксировать: BRD говорит, что даже шаблонный BR подписывается wet-signature → скан → DSC/eSign-аттестация; у нас BR подписывается только электронной DSC в портале. Это осознанное упрощение «всё в портале», но стоит назвать явно.
