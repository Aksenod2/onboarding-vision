> Out of scope: DVU-ветка, загрузка своего BR, крайние случаи, реальный email — трек 3 / позже.
> Принцип: переиспользуем экраны Sole Proprietor где возможно; показываем целевое видение ([[v2-target-vision]]).

## Роли в сценарии — МУЛЬТИРОЛЬ (набор, не одно значение)
Роль участника = **набор** ролей (`roles: Role[]`), одно лицо может нести несколько. Подтверждено BRD ([[v2-roles-model]]).
- **Customer Representative** — заполнитель/инициатор заявки. Готовит данные, рассылает ссылки, видит дашборд-монитор. **Может сам быть подписантом, а может не быть.**
- **Director** — директор (подписывает BR; мин. 2 для Company).
- **Authorized Signatory (AS)** — распоряжается счётом; по BRD = один из директоров.
- **UBO** — бенефициар (>10%).

Идентификацию (Aadhaar+видео+подпись) проходит тот, у кого есть роль подписанта (Director/AS). Если Customer Representative сам подписант — он и заполняет, и проходит свою сессию; если нет — только заполняет.

## Развилка Sole Proprietor ↔ Компания
В **технической навигации** (карта экранов ☰), не в клиентском UI. Две точки входа для демо: «Sole Proprietor» и «Компания». Клиенту тип бизнеса не выбираем (для прототипа).

## Фаза A — Заполнитель готовит заявку

| Шаг | Экран | Статус |
|---|---|---|
| Вход | Лендинг → Письмо → Регистрация | ✅ переиспуем |
| 1 | Доступ к реестрам + **PAN компании** | ✅ экран PAN, данные компании |
| 2 | **Бизнес-анкета + блок BR-вопросов** | 🔶 анкета есть, + новый блок BR |
| 3 | Подтверждение данных компании | ✅ переиспуем |
| 4 | **Рассылка ссылок подписантам** | 🆕 новый экран |

Заполнитель НЕ проходит Aadhaar / видео / подписание — это делают подписанты в фазе B.

### Блок BR-вопросов (в анкете; BRD §6 пп.1-4,7)
1. Свой BR или шаблон банка? → happy = **шаблон банка** (генерируется)
2. PAN компании → авто: имя компании, адрес
3. Мин. 2 директора-подписанта → дропдаун из Probe (имя, должность) + email/телефон вручную
4. Назначить Authorized Signatory → вручную (имя, email, телефон)
7. Подтвердить сгенерированный BR → кнопка (таймстамп)

### Экран рассылки (шаг 4)
«Ссылки-приглашения отправлены подписантам» + список троих. CTA → дашборд компании.

## Дашборд компании (заполнителя) — статус-монитор

Заполнитель видит по КАЖДОМУ подписанту: на каком шаге он сейчас, прошёл ли видео, подписал ли. Пример:

```
Заявка компании «…» — В обработке

Подписанты:
• Rajesh Kumar (директор)      ✅ Подписано
• Priya Sharma (директор)      ⏳ Видеоидентификация   [Войти как Priya]
• Amit Patel (Authorized Sig.) ○ Ожидает               [Войти как Amit]

Счёт откроется, когда все подписанты пройдут идентификацию и подпишут.
```

- Статус подписанта = его текущий шаг мини-сессии (Ожидает / Согласия / Aadhaar / Видео / Подписание / Готово).
- **«Войти как [имя]»** (вариант А) → имитация перехода по ссылке из письма → его персональная сессия.
- Все подписанты ✅ → блок «Счёт открыт» (как у Sole Proprietor).
