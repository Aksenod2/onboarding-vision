## Базовые решения v2

- **Роль:** Sole Proprietor. Подписантов нет (владелец сам себе подписант) → выбор директоров / authorized signatory / мульти-VCIP **не показываем**.
- **Вопрос «роль в компании» убран** — для Sole Prop роль выводится из PAN (4-й символ `P` = физлицо; подтверждено офиц. источниками). PAN предзаполнен примером `ABFPS4321K`, тип «Sole Proprietorship» показывается бейджем на шаге PAN.
- **Языки:** по умолчанию **русский**, локализация **RU / EN**, переключатель **сквозной** (общий `LanguageContext`) — на лендинге и в шапке всех экранов.
- **Риск-скор** (BNQ) — внутренний, клиенту **не показываем**.
- **Дашборд — дом заявки** (не финал): живая карта статусов с «Requires Action».
- **Happy-path / STP** — данные подтянуты из Probe42; ручной ввод — fallback («data not available»).
- **Режим свободной проверки:** CTA-кнопки не блокируются, любой шаг доступен (для дизайн-ревью). ☰-меню (справа внизу) — секция «Онбординг v2».

## Модель навигации — дашборд-хаб

После регистрации клиент попадает не в линейный конвейер, а на **дашборд**. Дашборд = карта шагов со статусами; клик по шагу → экран шага → возврат на дашборд (шаг помечается выполненным).

- **Логотип «SBER Банк»** (в шапке) → возврат на лендинг `/v2`.
- **«← Обзор заявки»** (в шапке шагов) → дашборд `/v2/dashboard`.
- **Статусы шагов:** ✓ done (кликабелен — посмотреть/править) · ● current · ○ pending · ⚠ requires_action (акцент + действие).
- **Финал** — не отдельный экран: когда все шаги done, дашборд показывает блок «Счёт открыт».
- Реестр шагов и роутов — `app/src/ui/v2/steps.ts` (единый источник для дашборда и навигации).

```
Лендинг → Письмо → Регистрация
                       ↓
            ┌──── ДАШБОРД (дом заявки) ────┐
            │ 1 Согласие на реестры        │
            │ 2 PAN и проверки             │  клик по шагу
            │ 3 Данные компании            │   ↓
            │ 4 Согласия по данным         │  экран шага
            │ 5 Бизнес-анкета (BNQ)        │   ↓
            │ 6 Согласие перед видео       │  возврат на дашборд
            │ 7 Видео-ID и подписание      │
            └──────────────────────────────┘
                       ↓ всё done
            Дашборд: «Счёт открыт ✓»
```

## Маршрут и экраны

### Пре-хаб (линейно)
**SP-01 Лендинг** (`/v2`) — hero «Banking built for India–Russia trade flows», CTA «Стать клиентом / Become our customer», переключатель RU/EN. Логотип «SBER Банк». → письмо.
**SP-02 Email-приглашение** (`/v2/email`) — письмо со ссылкой (иллюстративно). → регистрация.
**SP-03 Регистрация** (`/v2/login`) — email + телефон → OTP (демо-код `0000`); согласия Cookie (1), T&C 2b (2b, Y), Privacy Notice (3) с timestamp IST. → **дашборд**.

### Шаги хаба (порядок по [[BNQ Template A — Sole Proprietorship]] + транскрипт Марго)
1. **Согласие на реестры** (`/v2/registry`, id `registry`) — согласие на запрос данных из реестров **до PAN** (BR-01). → дашборд.
2. **PAN и проверки** (`/v2/pan`, id `pan`) — PAN предзаполнен `ABFPS4321K`, бейдж «Тип: Sole Proprietorship». Авто-скрининг: PAN, OFAC, Probe42, CKYC, **Стоп-42** (лицо). → дашборд.
3. **Данные компании** (`/v2/company`, id `company`) — Probe42-данные, бейджи «из реестра»; режим **Edit + proof-upload** → DVU, счёт заморожен (BR-05). → дашборд.
4. **Согласия по данным** (`/v2/data-consents`, id `data-consents`) — **единый блок** Aadhaar (6), Data Principals (5), KMP Confirmation (4) **перед анкетой** (по транскрипту Марго). → дашборд.
5. **Бизнес-анкета BNQ** (`/v2/bnq`, id `bnq`) — Q1–Q11, двойной сценарий «Probe42 подтянул / нет», ветвление ВЭД (Q9), upload business licence (Q1) / IEC (Q11). Риск-скор не показываем. Согласий внутри больше нет (вынесены в шаг 4). → дашборд.
6. **Согласие перед видео** (`/v2/pre-vcip`, id `pre-vcip`) — privacy notice + достоверность данных (BR-02, Consent 7). → дашборд.
7. **Видео-ID и подписание** (`/v2/vcip`, id `vcip`) — VCIP (один владелец) + подписание декларации/CAF в сессии. → дашборд.

### Хаб
**Дашборд** (`/v2/dashboard`) — карта 7 шагов со статусами, «Что требуется от вас», провал в любой шаг, блок «Счёт открыт» при завершении.

## Что отложено (до других ролей)

Появится при добавлении **Private Limited** и др. типов:
- Реальный **выбор роли/типа юрлица** (сейчас предвыбран Sole Prop).
- Выбор директоров-подписантов из Probe42-дропдауна (мин. 2 директора / 1 секретарь).
- Выбор / добавление authorized signatory + права.
- Мульти-VCIP (каждый подписант отдельно).
- Полноценный Board Resolution как документ.
- Вариативность BNQ Template B/C по типу конституции.
